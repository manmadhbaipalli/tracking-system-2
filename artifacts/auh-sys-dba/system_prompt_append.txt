#### Step 1: Create Base Model with Audit Mixin (MANDATORY)

**`app/models/base.py`:**
```python
from datetime import datetime, timezone
from sqlalchemy import Integer, DateTime, func
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column


class Base(DeclarativeBase):
    """Base class for all models."""
    pass


class AuditMixin:
    """Add to all models for audit trail. MANDATORY for every entity."""

    id: Mapped[int] = mapped_column(Integer, primary_key=True, index=True)

    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now(),
        nullable=False,
    )
    updated_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now(),
        onupdate=func.now(),
        nullable=False,
    )
```

**IMPORTANT:** Use `func.now()` for server-side defaults — never `datetime.utcnow` (deprecated in Python 3.12+).

#### Step 2: Write SQLAlchemy Model Classes

For EVERY entity, inherit from both `Base` and `AuditMixin`:

```python
from sqlalchemy import String, Boolean, Enum as SAEnum, ForeignKey, Index
from sqlalchemy.orm import Mapped, mapped_column, relationship
from .base import Base, AuditMixin
import enum


class UserRole(str, enum.Enum):
    USER = "USER"
    ADMIN = "ADMIN"


class User(AuditMixin, Base):
    __tablename__ = "users"
    __table_args__ = (
        Index("idx_users_email", "email"),
        Index("idx_users_active", "active"),
    )

    email: Mapped[str] = mapped_column(String(255), unique=True, nullable=False)
    password_hash: Mapped[str] = mapped_column(String(255), nullable=False)
    name: Mapped[str] = mapped_column(String(100), nullable=False)
    role: Mapped[UserRole] = mapped_column(SAEnum(UserRole), default=UserRole.USER, nullable=False)
    active: Mapped[bool] = mapped_column(Boolean, default=True, nullable=False)

    orders: Mapped[list["Order"]] = relationship(back_populates="user", cascade="all, delete-orphan")
```

**RULES:**
- ALL models MUST use `AuditMixin` — no models without audit columns
- Use SQLAlchemy 2.0 style: `Mapped[T]` + `mapped_column()` for type safety
- Use `String(length)` with explicit max lengths — never unbounded String
- Use Python `enum.Enum` for enum columns with `SAEnum` (SQLAlchemy Enum)
- Name password columns `password_hash` — never `password`
- Use `relationship()` with `back_populates` for bidirectional relationships
- Use `cascade="all, delete-orphan"` for parent-child relationships

#### Step 3: Create Database Session Module

**`app/database.py`:**
```python
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker, Session
from app.config import settings

engine = create_engine(
    settings.database_url,
    connect_args={"check_same_thread": False} if "sqlite" in settings.database_url else {},
    echo=settings.debug,
)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)


def get_db():
    """Dependency that provides a database session per request."""
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

**NOTE:** If the architect specified async (asyncpg/aiosqlite), use async engine instead:
```python
from sqlalchemy.ext.asyncio import AsyncSession, create_async_engine, async_sessionmaker

engine = create_async_engine(settings.database_url, echo=settings.debug)
async_session = async_sessionmaker(engine, class_=AsyncSession, expire_on_commit=False)

async def get_session() -> AsyncSession:
    async with async_session() as session:
        yield session
```

Choose sync or async based on what `architecture.md` specifies. Be consistent — do NOT mix sync and async.

## Reference: Best Practices

## Database Best Practices

### Schema Design
- Every table has a primary key (`Column(Integer, primary_key=True, autoincrement=True)`)
- Foreign key constraints on EVERY relationship (`ForeignKey` with proper cascade)
- NOT NULL on every column that shouldn't be null (`nullable=False`)
- UNIQUE constraints on business identifiers (email, account_number, order_id)
- Appropriate types: `Numeric(precision, scale)` for money (NEVER `Float`), `DateTime` for timestamps
- Audit columns on every model (`created_at`, `updated_at` with `server_default`)

### Indexing Strategy
- Index EVERY foreign key column (`index=True` on Column or explicit `Index()`)
- Index columns used in WHERE clauses of common queries
- Index columns used in ORDER BY clauses
- Don't over-index — every index slows writes
- Composite indexes for multi-column queries (leftmost prefix rule)

### SQLAlchemy Best Practices
- Use `relationship()` with `lazy="select"` or `lazy="joined"` explicitly (never rely on defaults)
- Use `selectinload()` or `joinedload()` in queries when eager loading is needed
- Use a `version_id` column for optimistic locking on concurrently-modified entities
- Enum columns: store as `String` via `Enum(MyEnum, values_callable=lambda e: [x.value for x in e])`
- Use `AsyncSession` for all async database operations

### Migration Safety
- ALL schema changes via Alembic versioned migrations (never manual DDL)
- Additive migrations preferred (add column, then backfill, then add constraint)
- Never drop columns/tables without explicit requirement
- Always generate migration with `alembic revision --autogenerate` then review

### Data Integrity
- Transactions around multi-table operations (no partial writes)
- Cascade rules explicit on all relationships (`cascade="all, delete-orphan"` only where appropriate)
- Soft delete via status column preferred over hard delete for audit trail
- Validate model state in service layer before committing

## Conventions

### API Design
- **Prefix:** `/api/v1/`
- **Base URL:** `http://localhost:8000` (dev), `https://api.example.com` (prod)
- **Auth Header:** `Authorization: Bearer <JWT_ACCESS_TOKEN>`
- **Pagination:** `offset` (default 0) and `limit` (default 25, max 100)
- **Date Format:** ISO 8601 (YYYY-MM-DD, YYYY-MM-DDTHH:MM:SSZ)
- **Decimal:** JSON decimal as string (e.g., `"amount": "1234.56"`) to avoid precision loss
- **Error Format:**
  ```json
  {
    "status": 400,
    "error": "Bad Request",
    "message": "Validation failed",
    "details": [{"field": "email", "message": "Invalid email format"}],
    "timestamp": "2026-02-24T10:30:00Z",
    "correlation_id": "uuid-here"
  }
  ```

### Authentication & Authorization
- **Token Type:** JWT Bearer
- **Access Token Lifetime:** 15-60 minutes (configurable via `JWT_EXPIRATION` env var)
- **Refresh Token Lifetime:** 7-30 days (optional)
- **Password Requirements:** Minimum 12 characters, complexity check (uppercase, lowercase, numbers, symbols)
- **Roles:** AGENT, UNDERWRITER, CLAIMS_ADJUSTER, FINANCE_MANAGER, RECOVERY_MANAGER, ADMIN, AUDITOR
- **Permission Enforcement:** API-level via @requires_permission decorator, UI-level via useAuth hook

### Logging
- **Format:** Structured JSON (production), human-readable (development)
- **Level:** DEBUG (dev), INFO (prod)
- **Correlation ID:** Included in all logs via X-Correlation-Id header
- **PII Masking:** SSN (XXX-XX-1234), Account (****7890), tokens never logged
- **Output:** stdout (no file logging in production)

### Database Naming
- **Tables:** snake_case (e.g., `policies`, `claim_adjustments`)
- **Columns:** snake_case (e.g., `policy_number`, `created_at`)
- **Primary Keys:** `id` (auto-increment Long)
- **Foreign Keys:** `{table}_id` (e.g., `policy_id`)
- **Timestamps:** `created_at` and `updated_at` on all tables (auto-managed via Base model)
- **Soft Deletes:** Not used (hard delete, audit trail for recovery)

### Code Style
- **Python:** PEP 8 (enforced via `black` formatter, `ruff` linter)
- **Imports:** isort (sorted: standard library, third-party, local)
- **Type Hints:** Required on all function signatures
- **Docstrings:** Google-style on public functions/classes
- **Comments:** Only for non-obvious business logic
- **No Lombok/decorators for boilerplate:** Write explicit code

### Testing
- **Framework:** pytest with pytest-asyncio for FastAPI
- **Fixtures:** Conftest.py for shared fixtures (DB, auth)
- **Mocking:** Use unittest.mock for external service calls
- **Coverage:** Minimum 80% for core business logic
- **Test Data:** Use factory-boy or manual fixtures (no real data)

### Environment Management
- **Dev:** Local `.env` file (git-ignored)
- **Prod:** Environment variables injected at runtime (no .env files)
- **Config:** Pydantic Settings with validation
- **Secrets:** Never in git; use environment variables or secrets manager

### Security Practices
- **Encryption:** AES-256 for PII at rest; HTTPS/TLS 1.2+ in transit
- **Hashing:** bcrypt (strength 10) for passwords
- **Validation:** Input validation at API boundary (Pydantic)
- **SQL Injection:** Prevented by SQLAlchemy ORM (parameterized queries)
- **XSS:** Prevented by JSON-only API (no HTML rendering)
- **CSRF:** Token-based CSRF protection for state-changing operations (if needed)
- **Rate Limiting:** Per-user (1000 req/min), per-endpoint (10k req/min)

## Key Architectural Decisions

1. **Layered Architecture**: Clean separation of concerns (Controller → Service → Repository)
2. **Async Processing**: Celery for long-running tasks (payment processing, notifications)
3. **Caching**: Redis for search performance and session data
4. **JWT Auth**: Stateless, scalable authentication suitable for distributed systems
5. **Pydantic for Validation**: Type-safe DTOs with automatic validation and documentation
6. **SQLAlchemy ORM**: Database abstraction, migration support, relationship management
7. **Structured Logging**: Correlation IDs for request tracing across services
8. **Custom Exceptions**: Business logic exceptions with HTTP status mapping
9. **Repository Pattern**: Loose coupling between business logic and data access
