#### Step 2: Create Multi-Stage Dockerfile

Create a two-stage Dockerfile:

**Stage 1: Build (Node.js)**
```dockerfile
FROM node:20-alpine AS build
WORKDIR /app

# Install dependencies first (cache layer)
COPY package.json package-lock.json ./
RUN npm ci --no-audit --no-fund

# Copy source and build
COPY . .

# Accept build-time environment variables
ARG VITE_API_BASE_URL
ENV VITE_API_BASE_URL=$VITE_API_BASE_URL

RUN npm run build
```

**Stage 2: Serve (nginx:alpine)**
```dockerfile
FROM nginx:alpine AS production

# Copy custom nginx configuration
COPY nginx.conf /etc/nginx/conf.d/default.conf

# Copy built assets from build stage
COPY --from=build /app/dist /usr/share/nginx/html

# Create non-root user
RUN addgroup -g 1001 -S appgroup && \
    adduser -S appuser -u 1001 -G appgroup && \
    chown -R appuser:appgroup /usr/share/nginx/html && \
    chown -R appuser:appgroup /var/cache/nginx && \
    chown -R appuser:appgroup /var/log/nginx && \
    touch /var/run/nginx.pid && \
    chown -R appuser:appgroup /var/run/nginx.pid

USER appuser

EXPOSE 80

HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:80/health || exit 1

CMD ["nginx", "-g", "daemon off;"]
```

**Key rules:**
- `npm ci` (not `npm install`) for reproducible builds
- Copy `package.json` + `package-lock.json` BEFORE source for dependency cache layer
- Build-time environment variables passed via `ARG` (Vite bakes them into the bundle)
- Non-root user in the runtime stage
- HEALTHCHECK for container orchestration

#### Step 3: Create nginx.conf

Create `nginx.conf` with SPA routing support:

```nginx
server {
    listen 80;
    server_name _;
    root /usr/share/nginx/html;
    index index.html;

    # Health check endpoint
    location /health {
        access_log off;
        return 200 '{"status":"UP"}';
        add_header Content-Type application/json;
    }

    # SPA routing -- try the file, then the directory, then fall back to index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Cache static assets aggressively (JS, CSS, images with content hash)
    location ~* \.(?:css|js|map|jpe?g|png|gif|ico|svg|woff2?|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    # Do not cache index.html (it contains references to hashed assets)
    location = /index.html {
        expires -1;
        add_header Cache-Control "no-store, no-cache, must-revalidate";
    }

    # Enable gzip compression
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types
        text/plain
        text/css
        text/javascript
        application/javascript
        application/json
        application/xml
        image/svg+xml;

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
}
```

**CRITICAL:** The `try_files $uri $uri/ /index.html` directive is ESSENTIAL for SPA routing. Without it, refreshing the browser on any route other than `/` will return a 404 because nginx looks for a physical file matching the URL path.

## Reference: Best Practices

## DevOps Best Practices for React Applications

### Multi-Stage Docker Builds

- [ ] **Stage 1 (build):** Use `node:20-alpine` for smallest build image
- [ ] **Stage 2 (serve):** Use `nginx:alpine` for smallest runtime image (~40MB vs ~1GB with node_modules)
- [ ] Copy `package.json` + `package-lock.json` BEFORE source code to leverage Docker layer caching for dependencies
- [ ] Use `npm ci` (not `npm install`) for deterministic, reproducible dependency resolution
- [ ] Pass `--no-audit --no-fund` to `npm ci` to speed up CI builds
- [ ] Final image should contain ONLY the `dist/` output and nginx config -- no source code, no node_modules, no build tools
- [ ] Tag images with both `latest` and the git commit SHA for traceability

### nginx SPA Routing

- [ ] **`try_files $uri $uri/ /index.html` is MANDATORY.** This is the single most important nginx directive for any SPA. Without it, refreshing the browser on `/dashboard` returns a 404 because nginx looks for a file named `dashboard` on disk.
- [ ] The `/index.html` fallback tells nginx: "if no physical file matches the URL, serve index.html and let React Router handle the route"
- [ ] Test by deploying and refreshing the browser on a nested route (e.g., `/users/123/edit`) -- it must render the correct page, not a 404
- [ ] Create a dedicated `/health` endpoint in nginx that returns JSON (not a redirect to index.html) for container health checks

### Gzip Compression

- [ ] Enable `gzip on` in nginx for text-based content
- [ ] Compress these types: `text/plain`, `text/css`, `text/javascript`, `application/javascript`, `application/json`, `application/xml`, `image/svg+xml`
- [ ] Set `gzip_min_length 1024` to avoid compressing tiny files (overhead exceeds benefit)
- [ ] Set `gzip_comp_level 6` for good compression-to-CPU ratio
- [ ] Enable `gzip_vary on` so proxies cache compressed and uncompressed versions separately
- [ ] Consider pre-compressing files at build time (`gzip -k dist/**/*.js`) for even better performance

### Cache Headers for Static Assets

- [ ] **Hashed assets (JS, CSS, images):** Set `Cache-Control: public, immutable` with `expires 1y`. Vite generates content-hashed filenames (`app.a1b2c3d4.js`), so when the content changes, the filename changes -- making long cache lifetimes safe.
- [ ] **index.html:** Set `Cache-Control: no-store, no-cache, must-revalidate` with `expires -1`. This file contains `<script>` tags pointing to hashed filenames, so the browser must always fetch the latest version to pick up new assets.
- [ ] Disable access logging for static asset requests to reduce log noise: `access_log off`
- [ ] Set `add_header X-Content-Type-Options "nosniff"` to prevent MIME type sniffing

### Environment Variables at Build Time (VITE_*)

- [ ] All environment variables accessible in client-side code must be prefixed with `VITE_` (Vite requirement)
- [ ] Vite replaces `import.meta.env.VITE_*` with literal values at build time -- they are baked into the JavaScript bundle
- [ ] This means environment variables CANNOT be changed at runtime -- you must rebuild the Docker image for different environments
- [ ] Use Docker `ARG` to pass build-time variables: `ARG VITE_API_BASE_URL` then `ENV VITE_API_BASE_URL=$VITE_API_BASE_URL`
- [ ] In docker-compose, pass build args: `build: { args: { VITE_API_BASE_URL: ${VITE_API_BASE_URL} } }`
- [ ] In CI/CD, pass as build-args to `docker build --build-arg VITE_API_BASE_URL=...`
- [ ] NEVER put secrets (API keys, tokens) in `VITE_*` variables -- they end up in the JavaScript bundle visible to anyone
- [ ] Provide `.env.example` with placeholder values for all required variables

### Health Check Endpoint

- [ ] Create a `/health` endpoint in nginx that returns `{"status": "UP"}` with `Content-Type: application/json`
- [ ] This endpoint must be lightweight: `return 200` with `access_log off`
- [ ] Dockerfile `HEALTHCHECK` should use `wget` (available in alpine) to check this endpoint
- [ ] Set `--start-period=5s` for nginx (starts fast, unlike a JVM application)
- [ ] Set `--interval=30s --timeout=3s --retries=3` for reasonable failure detection
- [ ] In docker-compose, mirror the same health check configuration
- [ ] The health endpoint must NOT be behind authentication or redirect to index.html

### CI Caching of node_modules

- [ ] Use `actions/setup-node@v4` with `cache: 'npm'` -- this automatically caches `~/.npm` based on `package-lock.json` hash
- [ ] Use `npm ci` (not `npm install`) in CI -- it's faster and ensures deterministic installs from lockfile
- [ ] For Docker builds, use GitHub Actions cache: `cache-from: type=gha` and `cache-to: type=gha,mode=max`
- [ ] Cache the Docker build layers to avoid rebuilding unchanged dependency layers
- [ ] Upload build artifacts with `actions/upload-artifact` for downstream jobs (e.g., deploy)

### Security Best Practices

- [ ] Run nginx as non-root user in the Dockerfile
- [ ] Add security headers in nginx: `X-Frame-Options`, `X-Content-Type-Options`, `X-XSS-Protection`, `Referrer-Policy`
- [ ] Consider adding `Content-Security-Policy` header (may need tuning for inline styles/scripts)
- [ ] `.dockerignore` must exclude `.env`, `.git`, `node_modules`, and any sensitive files
- [ ] Never commit `.env` files -- only `.env.example` with placeholder values
- [ ] NEVER store secrets in `VITE_*` environment variables (they are embedded in the JS bundle and visible in the browser)
- [ ] Use HTTPS in production (configure SSL termination at the load balancer or reverse proxy level)

### Production Optimization

- [ ] Vite tree-shakes unused code automatically -- ensure no circular imports that break tree-shaking
- [ ] Enable source maps only in staging, not production (prevent code exposure)
- [ ] Set `NODE_ENV=production` during the build stage
- [ ] Analyze bundle size with `npx vite-bundle-visualizer` and address any unexpectedly large chunks
- [ ] Consider splitting vendor chunks: configure `build.rollupOptions.output.manualChunks` in vite.config.ts

## Conventions

### API Design
- **Prefix:** `/api/v1/`
- **Base URL:** `http://localhost:8000` (dev), `https://api.example.com` (prod)
- **Auth Header:** `Authorization: Bearer <JWT_ACCESS_TOKEN>`
- **Pagination:** `offset` (default 0) and `limit` (default 25, max 100)
- **Date Format:** ISO 8601 (YYYY-MM-DD, YYYY-MM-DDTHH:MM:SSZ)
- **Decimal:** JSON decimal as string (e.g., `"amount": "1234.56"`) to avoid precision loss
- **Error Format:**
  ```json
  {
    "status": 400,
    "error": "Bad Request",
    "message": "Validation failed",
    "details": [{"field": "email", "message": "Invalid email format"}],
    "timestamp": "2026-02-24T10:30:00Z",
    "correlation_id": "uuid-here"
  }
  ```

### Authentication & Authorization
- **Token Type:** JWT Bearer
- **Access Token Lifetime:** 15-60 minutes (configurable via `JWT_EXPIRATION` env var)
- **Refresh Token Lifetime:** 7-30 days (optional)
- **Password Requirements:** Minimum 12 characters, complexity check (uppercase, lowercase, numbers, symbols)
- **Roles:** AGENT, UNDERWRITER, CLAIMS_ADJUSTER, FINANCE_MANAGER, RECOVERY_MANAGER, ADMIN, AUDITOR
- **Permission Enforcement:** API-level via @requires_permission decorator, UI-level via useAuth hook

### Logging
- **Format:** Structured JSON (production), human-readable (development)
- **Level:** DEBUG (dev), INFO (prod)
- **Correlation ID:** Included in all logs via X-Correlation-Id header
- **PII Masking:** SSN (XXX-XX-1234), Account (****7890), tokens never logged
- **Output:** stdout (no file logging in production)

### Database Naming
- **Tables:** snake_case (e.g., `policies`, `claim_adjustments`)
- **Columns:** snake_case (e.g., `policy_number`, `created_at`)
- **Primary Keys:** `id` (auto-increment Long)
- **Foreign Keys:** `{table}_id` (e.g., `policy_id`)
- **Timestamps:** `created_at` and `updated_at` on all tables (auto-managed via Base model)
- **Soft Deletes:** Not used (hard delete, audit trail for recovery)

### Code Style
- **Python:** PEP 8 (enforced via `black` formatter, `ruff` linter)
- **Imports:** isort (sorted: standard library, third-party, local)
- **Type Hints:** Required on all function signatures
- **Docstrings:** Google-style on public functions/classes
- **Comments:** Only for non-obvious business logic
- **No Lombok/decorators for boilerplate:** Write explicit code

### Testing
- **Framework:** pytest with pytest-asyncio for FastAPI
- **Fixtures:** Conftest.py for shared fixtures (DB, auth)
- **Mocking:** Use unittest.mock for external service calls
- **Coverage:** Minimum 80% for core business logic
- **Test Data:** Use factory-boy or manual fixtures (no real data)

### Environment Management
- **Dev:** Local `.env` file (git-ignored)
- **Prod:** Environment variables injected at runtime (no .env files)
- **Config:** Pydantic Settings with validation
- **Secrets:** Never in git; use environment variables or secrets manager

### Security Practices
- **Encryption:** AES-256 for PII at rest; HTTPS/TLS 1.2+ in transit
- **Hashing:** bcrypt (strength 10) for passwords
- **Validation:** Input validation at API boundary (Pydantic)
- **SQL Injection:** Prevented by SQLAlchemy ORM (parameterized queries)
- **XSS:** Prevented by JSON-only API (no HTML rendering)
- **CSRF:** Token-based CSRF protection for state-changing operations (if needed)
- **Rate Limiting:** Per-user (1000 req/min), per-endpoint (10k req/min)

## Key Architectural Decisions

1. **Layered Architecture**: Clean separation of concerns (Controller → Service → Repository)
2. **Async Processing**: Celery for long-running tasks (payment processing, notifications)
3. **Caching**: Redis for search performance and session data
4. **JWT Auth**: Stateless, scalable authentication suitable for distributed systems
5. **Pydantic for Validation**: Type-safe DTOs with automatic validation and documentation
6. **SQLAlchemy ORM**: Database abstraction, migration support, relationship management
7. **Structured Logging**: Correlation IDs for request tracing across services
8. **Custom Exceptions**: Business logic exceptions with HTTP status mapping
9. **Repository Pattern**: Loose coupling between business logic and data access
