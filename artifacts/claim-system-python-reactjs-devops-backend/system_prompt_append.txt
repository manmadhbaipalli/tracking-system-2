## Dockerfile Template (Multi-Stage, Non-Root)

```dockerfile
# Build stage
FROM python:3.12-slim AS builder
WORKDIR /app
RUN apt-get update && apt-get install -y --no-install-recommends gcc libpq-dev && rm -rf /var/lib/apt/lists/*
COPY requirements.txt .
RUN pip install --no-cache-dir --prefix=/install -r requirements.txt

# Runtime stage
FROM python:3.12-slim
WORKDIR /app
RUN apt-get update && apt-get install -y --no-install-recommends libpq5 curl && rm -rf /var/lib/apt/lists/*
RUN groupadd -r appgroup && useradd -r -g appgroup -d /app appuser
COPY --from=builder /install /usr/local
COPY . .
RUN chown -R appuser:appgroup /app
USER appuser
EXPOSE 8000
HEALTHCHECK --interval=30s --timeout=5s --start-period=30s --retries=3 \
  CMD curl -f http://localhost:8000/health/live || exit 1
CMD ["gunicorn", "app.main:app", "-c", "gunicorn.conf.py"]
```

## Reference: Best Practices

## DevOps Best Practices

### Container Security
- Multi-stage builds (separate builder with gcc/build tools + slim runtime)
- Non-root user (never run as root in production)
- Pin base image versions (`python:3.12-slim`, not `python:latest`)
- No secrets in Docker image (no COPY .env, no ARG for secrets)
- Minimal runtime image (no build tools — `--no-cache-dir` on pip for smaller layers)
- .dockerignore excludes .git, .env, .venv, __pycache__, *.md, docker-compose.yml

### CI/CD Pipeline
- Pipeline order: lint -> test -> security scan -> build -> deploy
- All tests must pass before build (no "deploy anyway")
- Dependency vulnerability scanning (pip-audit for known CVEs)
- SAST scanning (bandit for Python security analysis)
- Secret scanning in CI (detect accidentally committed secrets)
- Build artifacts are immutable (same image in staging and production)

### 12-Factor Deployment
- Config via environment variables (never baked into image)
- Required config validated on startup via Pydantic Settings (fail fast with clear error)
- Logs to stdout as structured JSON (not files — container runtime captures stdout)
- Health checks on `/health/live` and `/health/ready` endpoints
- Graceful shutdown handling (gunicorn with `--graceful-timeout`)
- Resource limits set (CPU, memory) on all containers

### Reliability
- Health checks on ALL services in docker-compose
- `depends_on` with `condition: service_healthy`
- Restart policy: `unless-stopped`
- Named volumes for persistent data (never bind mounts in production)
- gunicorn with UvicornWorker — workers based on CPU count (`2*CPU+1`)

## GitHub Actions Pipeline: `.github/workflows/ci.yml`

```yaml
name: CI/CD Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

permissions:
  contents: read
  security-events: write

jobs:
  # ============================================
  # Job 1: Lint and Type Check
  # ============================================
  lint:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"
          cache: pip

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install ruff mypy
          pip install -r requirements.txt

      - name: Lint with ruff
        run: ruff check app/

      - name: Type check with mypy
        run: mypy app/ --ignore-missing-imports || true

  # ============================================
  # Job 2: Test
  # ============================================
  test:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"
          cache: pip

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements.txt
          pip install pytest pytest-cov httpx

      - name: Run tests with coverage
        run: pytest -v --tb=short --cov=app --cov-report=xml

      - name: Upload coverage report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: coverage-report
          path: coverage.xml

  # ============================================
  # Job 3: Security Scanning
  # ============================================
  security-scan:
    runs-on: ubuntu-latest
    needs: test
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"
          cache: pip

      - name: Install dependencies
        run: pip install -r requirements.txt

      - name: Security scan with pip-audit (SCA)
        run: |
          pip install pip-audit
          pip-audit -r requirements.txt || true

      - name: Security scan with bandit (SAST)
        run: |
          pip install bandit
          bandit -r app/ -f json -o bandit-report.json || true

      - name: Upload security reports
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: security-reports
          path: bandit-report.json

  # ============================================
  # Job 4: Docker Build
  # ============================================
  docker-build:
    runs-on: ubuntu-latest
    needs: [test, security-scan]
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Build Docker image
        uses: docker/build-push-action@v5
        with:
          context: .
          push: false
          tags: app:latest
          cache-from: type=gha
          cache-to: type=gha,mode=max
```

### CI/CD Rules

- **4 jobs**: lint -> test -> security-scan -> docker-build
- **pip-audit** — checks for known vulnerabilities in Python packages (SCA)
- **bandit** — static analysis security testing for Python (SAST)
- **ruff** — fast Python linter (replaces flake8, isort, pyflakes)
- **mypy** — optional type checking (non-blocking with `|| true`)
- **Coverage report** — uploaded as artifact for tracking
- **Security reports** — bandit JSON output uploaded as artifact
- **Docker build only on main push** — not triggered on PRs
- **GHA cache** — speeds up subsequent Docker builds

## gunicorn.conf.py

```python
import multiprocessing
import os

# Bind
bind = f"0.0.0.0:{os.getenv('SERVER_PORT', '8000')}"

# Workers — 2 * CPU cores + 1 (async workers handle concurrency per worker)
workers = int(os.getenv("GUNICORN_WORKERS", multiprocessing.cpu_count() * 2 + 1))
worker_class = "uvicorn.workers.UvicornWorker"

# Timeouts
timeout = 120
graceful_timeout = 30
keepalive = 5

# Logging
accesslog = "-"
errorlog = "-"
loglevel = os.getenv("LOG_LEVEL", "info").lower()

# Security
limit_request_line = 8190
limit_request_fields = 100
limit_request_field_size = 8190

# Process naming
proc_name = "app"

# Preload app for faster worker startup
preload_app = True
```

### gunicorn Rules

- **`UvicornWorker`** — required for ASGI (FastAPI)
- **Workers based on CPU** — scalable
- **Preload app** — faster worker startup, shared memory
- **All values configurable via env vars** — no hardcoded values
- **Security limits** — prevent oversized requests

## Conventions

### API Design
- **Prefix:** `/api/v1/`
- **Base URL:** `http://localhost:8000` (dev), `https://api.example.com` (prod)
- **Auth Header:** `Authorization: Bearer <JWT_ACCESS_TOKEN>`
- **Pagination:** `offset` (default 0) and `limit` (default 25, max 100)
- **Date Format:** ISO 8601 (YYYY-MM-DD, YYYY-MM-DDTHH:MM:SSZ)
- **Decimal:** JSON decimal as string (e.g., `"amount": "1234.56"`) to avoid precision loss
- **Error Format:**
  ```json
  {
    "status": 400,
    "error": "Bad Request",
    "message": "Validation failed",
    "details": [{"field": "email", "message": "Invalid email format"}],
    "timestamp": "2026-02-24T10:30:00Z",
    "correlation_id": "uuid-here"
  }
  ```

### Authentication & Authorization
- **Token Type:** JWT Bearer
- **Access Token Lifetime:** 15-60 minutes (configurable via `JWT_EXPIRATION` env var)
- **Refresh Token Lifetime:** 7-30 days (optional)
- **Password Requirements:** Minimum 12 characters, complexity check (uppercase, lowercase, numbers, symbols)
- **Roles:** AGENT, UNDERWRITER, CLAIMS_ADJUSTER, FINANCE_MANAGER, RECOVERY_MANAGER, ADMIN, AUDITOR
- **Permission Enforcement:** API-level via @requires_permission decorator, UI-level via useAuth hook

### Logging
- **Format:** Structured JSON (production), human-readable (development)
- **Level:** DEBUG (dev), INFO (prod)
- **Correlation ID:** Included in all logs via X-Correlation-Id header
- **PII Masking:** SSN (XXX-XX-1234), Account (****7890), tokens never logged
- **Output:** stdout (no file logging in production)

### Database Naming
- **Tables:** snake_case (e.g., `policies`, `claim_adjustments`)
- **Columns:** snake_case (e.g., `policy_number`, `created_at`)
- **Primary Keys:** `id` (auto-increment Long)
- **Foreign Keys:** `{table}_id` (e.g., `policy_id`)
- **Timestamps:** `created_at` and `updated_at` on all tables (auto-managed via Base model)
- **Soft Deletes:** Not used (hard delete, audit trail for recovery)

### Code Style
- **Python:** PEP 8 (enforced via `black` formatter, `ruff` linter)
- **Imports:** isort (sorted: standard library, third-party, local)
- **Type Hints:** Required on all function signatures
- **Docstrings:** Google-style on public functions/classes
- **Comments:** Only for non-obvious business logic
- **No Lombok/decorators for boilerplate:** Write explicit code

### Testing
- **Framework:** pytest with pytest-asyncio for FastAPI
- **Fixtures:** Conftest.py for shared fixtures (DB, auth)
- **Mocking:** Use unittest.mock for external service calls
- **Coverage:** Minimum 80% for core business logic
- **Test Data:** Use factory-boy or manual fixtures (no real data)

### Environment Management
- **Dev:** Local `.env` file (git-ignored)
- **Prod:** Environment variables injected at runtime (no .env files)
- **Config:** Pydantic Settings with validation
- **Secrets:** Never in git; use environment variables or secrets manager

### Security Practices
- **Encryption:** AES-256 for PII at rest; HTTPS/TLS 1.2+ in transit
- **Hashing:** bcrypt (strength 10) for passwords
- **Validation:** Input validation at API boundary (Pydantic)
- **SQL Injection:** Prevented by SQLAlchemy ORM (parameterized queries)
- **XSS:** Prevented by JSON-only API (no HTML rendering)
- **CSRF:** Token-based CSRF protection for state-changing operations (if needed)
- **Rate Limiting:** Per-user (1000 req/min), per-endpoint (10k req/min)

### Development Setup
```bash

cd backend
python -m venv venv
```


## Key Architectural Decisions

1. **Layered Architecture**: Clean separation of concerns (Controller → Service → Repository)
2. **Async Processing**: Celery for long-running tasks (payment processing, notifications)
3. **Caching**: Redis for search performance and session data
4. **JWT Auth**: Stateless, scalable authentication suitable for distributed systems
5. **Pydantic for Validation**: Type-safe DTOs with automatic validation and documentation
6. **SQLAlchemy ORM**: Database abstraction, migration support, relationship management
7. **Structured Logging**: Correlation IDs for request tracing across services
8. **Custom Exceptions**: Business logic exceptions with HTTP status mapping
9. **Repository Pattern**: Loose coupling between business logic and data access
