```typescript
const api = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:8080/api',
  headers: {
    'Content-Type': 'application/json',
  },
});

// Request interceptor -- attach JWT token
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// Response interceptor -- handle 401 globally
api.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('token');
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);

export default api;
```

## Reference: Best Practices

## React Development Best Practices Checklist

### Component Design

- **Functional components ONLY.** Never use class components. All components must be function declarations or arrow functions with React.FC type (or explicit prop types).
- **Single Responsibility.** Each component does ONE thing. If a component file exceeds 200 lines, split it into smaller components.
- **Composition over inheritance.** Use children props, render props, or custom hooks to share behavior -- never inheritance.
- **Meaningful component names.** PascalCase, descriptive: `UserProfileCard`, `OrderHistoryTable`, `LoginForm`. Never `Component1`, `Item`, `Wrapper`.
- **Co-locate related files.** Keep component, its types, and its styles close together.

### TypeScript Strict Mode

- [ ] `strict: true` in tsconfig.json -- no exceptions
- [ ] `noUnusedLocals: true` -- catch dead code at compile time
- [ ] `noUnusedParameters: true` -- enforce clean function signatures
- [ ] Never use `any` type -- use `unknown` with type narrowing if type is truly unknown
- [ ] Define explicit return types for all exported functions and hooks
- [ ] Use `interface` for object shapes that may be extended, `type` for unions and intersections
- [ ] Use `as const` for literal types and configuration objects

### Custom Hooks for Shared Logic

- [ ] Extract repeated stateful logic into custom hooks (`useAuth`, `useFetch`, `useForm`, `useDebounce`)
- [ ] Custom hooks must start with `use` prefix (React rule)
- [ ] Hooks must return well-typed values -- never `any`
- [ ] Keep hooks focused on a single concern
- [ ] Avoid side effects in hooks that are hard to predict -- use `useEffect` cleanup properly
- [ ] Memoize expensive computations with `useMemo` and callback references with `useCallback`

### Error Boundaries

- [ ] Wrap major page sections in error boundaries to prevent full-app crashes
- [ ] Create a reusable `ErrorBoundary` component (this is the one case where a class component is acceptable -- React requires it)
- [ ] Error boundaries must show a user-friendly fallback UI, not a blank screen
- [ ] Log errors to a monitoring service in the error boundary's `componentDidCatch`
- [ ] Use `react-error-boundary` library for a functional API if preferred

### Loading and Error States

- [ ] EVERY component that fetches data must handle three states: loading, error, and success
- [ ] Use skeleton screens or spinners during loading -- never show a blank page
- [ ] Show user-friendly error messages -- never raw error objects or stack traces
- [ ] Handle empty states gracefully ("No items found" with an action to create one)
- [ ] Implement retry buttons for failed network requests
- [ ] Use `Suspense` with lazy-loaded components for code-split loading states

### Responsive Design

- [ ] Mobile-first approach: base styles for small screens, `md:` and `lg:` for larger
- [ ] Test at 320px, 768px, 1024px, and 1440px breakpoints minimum
- [ ] Use Tailwind CSS responsive prefixes consistently (`sm:`, `md:`, `lg:`, `xl:`)
- [ ] Navigation collapses to hamburger menu on mobile
- [ ] Tables become card layouts on small screens
- [ ] Touch targets are at least 44x44px for mobile
- [ ] No horizontal scroll on any screen size

### Accessibility Basics

- [ ] Semantic HTML elements: `<nav>`, `<main>`, `<section>`, `<article>`, `<button>`, `<header>`, `<footer>`
- [ ] All images have `alt` attributes (empty `alt=""` for decorative images)
- [ ] All form inputs have associated `<label>` elements (use `htmlFor`)
- [ ] Interactive elements are focusable and have visible focus indicators
- [ ] Color contrast meets WCAG AA (4.5:1 for text, 3:1 for large text)
- [ ] ARIA attributes used correctly when semantic HTML is insufficient
- [ ] Keyboard navigation works for all interactive elements (Tab, Enter, Escape)

### Key Props

- [ ] ALWAYS provide a stable `key` prop in lists (use unique ID, never array index)
- [ ] Keys must be unique among siblings -- not globally unique
- [ ] Never use `Math.random()` or `Date.now()` as keys
- [ ] If items can be reordered, index-based keys WILL cause bugs -- use item IDs

### Avoid Prop Drilling

- [ ] If a prop passes through 3+ levels, use React Context or a state management solution
- [ ] Create focused contexts: `AuthContext`, `ThemeContext`, `NotificationContext` -- not one giant `AppContext`
- [ ] Use custom hooks to access context: `useAuth()` instead of `useContext(AuthContext)`
- [ ] Consider component composition (passing children) before reaching for context
- [ ] For complex state, evaluate Zustand, Jotai, or Redux Toolkit -- but only when context becomes unwieldy

### Code Splitting with Lazy/Suspense

- [ ] Lazy-load page components: `const DashboardPage = lazy(() => import('./pages/DashboardPage'))`
- [ ] Wrap lazy components in `<Suspense fallback={<LoadingSpinner />}>`
- [ ] Split by route: each page is a separate chunk
- [ ] Do NOT lazy-load small, frequently-used components (Header, Button) -- only pages and heavy features
- [ ] Use meaningful chunk names with magic comments if using webpack: `import(/* webpackChunkName: "dashboard" */ ...)`

### State Management Rules

- [ ] Use `useState` for local component state
- [ ] Use `useReducer` for complex state logic with multiple sub-values
- [ ] Use Context for global state that changes infrequently (auth, theme, locale)
- [ ] Use a state library (Zustand, Redux Toolkit) only for complex shared state with frequent updates
- [ ] NEVER store derived data in state -- compute it during render or use `useMemo`
- [ ] Keep state as close to where it's used as possible (lift only when necessary)

### API Integration Patterns

- [ ] Centralize HTTP client configuration (base URL, interceptors, headers)
- [ ] Handle authentication token injection in a request interceptor -- not in every service call
- [ ] Handle 401 responses globally in a response interceptor (redirect to login)
- [ ] Use TypeScript generics in service functions: `getUsers(): Promise<User[]>`
- [ ] Never call `fetch` or `axios` directly in components -- always go through service modules
- [ ] Implement proper request cancellation with `AbortController` for unmounted components

### Performance

- [ ] Memoize expensive renders with `React.memo()` (only when profiling shows re-render issues)
- [ ] Use `useMemo` for expensive computations, `useCallback` for stable callback references passed to memoized children
- [ ] Avoid creating new objects/arrays in render (move to `useMemo` or outside the component)
- [ ] Debounce search input and other rapid-fire user actions
- [ ] Virtualize long lists (100+ items) with `react-window` or `@tanstack/virtual`
- [ ] Optimize images: use proper formats (WebP), lazy load with `loading="lazy"`, set explicit dimensions

## Conventions

### API Design
- **Prefix:** `/api/v1/`
- **Base URL:** `http://localhost:8000` (dev), `https://api.example.com` (prod)
- **Auth Header:** `Authorization: Bearer <JWT_ACCESS_TOKEN>`
- **Pagination:** `offset` (default 0) and `limit` (default 25, max 100)
- **Date Format:** ISO 8601 (YYYY-MM-DD, YYYY-MM-DDTHH:MM:SSZ)
- **Decimal:** JSON decimal as string (e.g., `"amount": "1234.56"`) to avoid precision loss
- **Error Format:**
  ```json
  {
    "status": 400,
    "error": "Bad Request",
    "message": "Validation failed",
    "details": [{"field": "email", "message": "Invalid email format"}],
    "timestamp": "2026-02-24T10:30:00Z",
    "correlation_id": "uuid-here"
  }
  ```

### Authentication & Authorization
- **Token Type:** JWT Bearer
- **Access Token Lifetime:** 15-60 minutes (configurable via `JWT_EXPIRATION` env var)
- **Refresh Token Lifetime:** 7-30 days (optional)
- **Password Requirements:** Minimum 12 characters, complexity check (uppercase, lowercase, numbers, symbols)
- **Roles:** AGENT, UNDERWRITER, CLAIMS_ADJUSTER, FINANCE_MANAGER, RECOVERY_MANAGER, ADMIN, AUDITOR
- **Permission Enforcement:** API-level via @requires_permission decorator, UI-level via useAuth hook

### Logging
- **Format:** Structured JSON (production), human-readable (development)
- **Level:** DEBUG (dev), INFO (prod)
- **Correlation ID:** Included in all logs via X-Correlation-Id header
- **PII Masking:** SSN (XXX-XX-1234), Account (****7890), tokens never logged
- **Output:** stdout (no file logging in production)

### Database Naming
- **Tables:** snake_case (e.g., `policies`, `claim_adjustments`)
- **Columns:** snake_case (e.g., `policy_number`, `created_at`)
- **Primary Keys:** `id` (auto-increment Long)
- **Foreign Keys:** `{table}_id` (e.g., `policy_id`)
- **Timestamps:** `created_at` and `updated_at` on all tables (auto-managed via Base model)
- **Soft Deletes:** Not used (hard delete, audit trail for recovery)

### Code Style
- **Python:** PEP 8 (enforced via `black` formatter, `ruff` linter)
- **Imports:** isort (sorted: standard library, third-party, local)
- **Type Hints:** Required on all function signatures
- **Docstrings:** Google-style on public functions/classes
- **Comments:** Only for non-obvious business logic
- **No Lombok/decorators for boilerplate:** Write explicit code

### Testing
- **Framework:** pytest with pytest-asyncio for FastAPI
- **Fixtures:** Conftest.py for shared fixtures (DB, auth)
- **Mocking:** Use unittest.mock for external service calls
- **Coverage:** Minimum 80% for core business logic
- **Test Data:** Use factory-boy or manual fixtures (no real data)

### Environment Management
- **Dev:** Local `.env` file (git-ignored)
- **Prod:** Environment variables injected at runtime (no .env files)
- **Config:** Pydantic Settings with validation
- **Secrets:** Never in git; use environment variables or secrets manager

### Security Practices
- **Encryption:** AES-256 for PII at rest; HTTPS/TLS 1.2+ in transit
- **Hashing:** bcrypt (strength 10) for passwords
- **Validation:** Input validation at API boundary (Pydantic)
- **SQL Injection:** Prevented by SQLAlchemy ORM (parameterized queries)
- **XSS:** Prevented by JSON-only API (no HTML rendering)
- **CSRF:** Token-based CSRF protection for state-changing operations (if needed)
- **Rate Limiting:** Per-user (1000 req/min), per-endpoint (10k req/min)

## Key Architectural Decisions

1. **Layered Architecture**: Clean separation of concerns (Controller → Service → Repository)
2. **Async Processing**: Celery for long-running tasks (payment processing, notifications)
3. **Caching**: Redis for search performance and session data
4. **JWT Auth**: Stateless, scalable authentication suitable for distributed systems
5. **Pydantic for Validation**: Type-safe DTOs with automatic validation and documentation
6. **SQLAlchemy ORM**: Database abstraction, migration support, relationship management
7. **Structured Logging**: Correlation IDs for request tracing across services
8. **Custom Exceptions**: Business logic exceptions with HTTP status mapping
9. **Repository Pattern**: Loose coupling between business logic and data access
